# WS6 Scaling LM Releases Expert

AI project manager for the WS6 Enabling Weekly Releases workstream (TomTom Q1/Q2 2026 Quality Sprint). Answers questions about WS6 status and generates executive-quality weekly reports using live data from Confluence, JIRA, and Slack.

## How to use

Open this repo in **Claude Code** (VS Code extension or CLI). The agent loads automatically from [CLAUDE.md](CLAUDE.md).

### Ask questions

Just type naturally. The agent fetches live data and answers in a structured, executive-friendly format:

```
What are the current blockers?
Who owns the idle time investigation?
Is CW18 on track?
What decisions were made last week?
```

### Generate the weekly report — `/report`

Fetches all live sources (Confluence, JIRA, Slack) and outputs the complete WS6 section for the next Workstreams Weekly Report, ready to review.

```
/report
```

What it does:
1. Finds the most recent weekly report in Confluence and reads the existing WS6 section
2. Pulls the last 50–100 messages from the three WS6 Slack channels
3. Fetches the key JIRA tickets
4. Generates the full WS6 section — Goal 1 delivery table (with the new week row added), Goal 2 risks, milestones, and key actions
5. Flags any gaps that need manual input (e.g. lead time figures from the master spreadsheet)

### Save to Confluence — `/save`

Pushes the report generated by `/report` directly to the Confluence draft page. Always run `/report` first.

```
/save
```

What it does:
- Overwrites the [Next Weekly Report Draft](https://tomtom.atlassian.net/wiki/spaces/ADPU/pages/1860600695/Next+Weekly+Report+Draft) page with the report from the current session
- Stamps a version message with the target week (e.g. "WS6 agent auto-save — W18")
- Confirms with a direct link on success

From the draft page, the team can review, add images/charts, fill in any remaining gaps, and copy the section into the official weekly report page.

## Setup

### Prerequisites

- [Claude Code](https://claude.ai/code) (VS Code extension or CLI)
- A claude.ai subscription (no API key needed — the agent runs through Claude Code)
- Atlassian OAuth (one-time, see below)
- Slack MCP connected (pre-configured)

### Atlassian authentication (first time only)

The Atlassian MCP is pre-configured in `~/.claude/settings.json`. On first use, Claude Code will prompt you to authorize via OAuth:

1. Open the authorization URL shown in the chat
2. Log in with your TomTom Atlassian account
3. After the browser redirects (the page may fail to load — that's fine), paste the full redirect URL back into the chat if prompted

Tokens are cached and refreshed automatically after that. You won't need to re-authorize unless tokens expire.

### Slack authentication

Slack is pre-configured in `~/.claude/settings.json` and connects automatically using your claude.ai Slack integration.
